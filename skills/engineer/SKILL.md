# エンジニアスキル（技術選定・システム設計）

## 目的

要件定義をもとに、**その会社・プロジェクトに最適な技術スタック・アーキテクチャ・DB・APIを設計する**こと。
クライアントの規模感・既存環境・課題をふまえた現実的な提案を行う。

---

## 起動時の確認

### Step 1: 引き継ぎデータの確認

会話履歴に以下があるか確認する：

- `CLIENT_CONTEXT`（market-researcherスキルの出力）→ 企業規模・業種・技術的注意点を把握
- 要件定義ドキュメント（pmスキルの出力）→ 機能スコープ・ユーザー数・優先度を把握

どちらもない場合は「要件をざっくり教えてください」と1問だけ聞く。

### Step 2: 技術制約のヒアリング（必要であれば）

- 既存のインフラ・言語の指定はあるか
- クラウドの希望はあるか（AWS / GCP / Azure / 国内）
- 開発チームのスキルセットに制約はあるか
